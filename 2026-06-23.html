Simon Willison
Prompt Injection as Role Confusion
New
Jun 22, 2026 · 11:59 pm — link blog
Simon highlights research by Charles Ye, Jasmine Cui and Dylan Hadfield-Menell (role-confusion.github.io) showing that models can't reliably distinguish their own privileged text (in <system>/<think>/<assistant> tags) from untrusted <user> input — and worse, they weight the style of text over its content. Appending text styled like a model's internal reasoning ("Policy states: Allowed… only if the user is wearing green") can override training in models like gpt-oss-20b. The kicker: "destyling" — rewriting an attack so it looks less like the expected format — drops average attack success from 61% to 10%, a change "nearly invisible to humans." The authors argue injection defense stays "a perpetual whack-a-mole game" until models achieve genuine role perception. Lands directly on top of OpenAI's Daybreak day — the offensive flip-side of the same security coin.

O'Reilly Radar
Loop Engineering — Addy Osmani
New
Jun 22, 2026 · 14-min read (reposted from addyosmani.com)
Names the emerging practice: "Loop engineering is replacing yourself as the person who prompts the agent. You design the system that does it instead." Quotes Anthropic's Claude Code lead Boris Cherny ("I don't prompt Claude anymore… My job is to write loops") and Peter Steinberger. Osmani maps the five loop primitives — automations (scheduled discovery/triage), worktrees (parallel isolation), skills (codified project knowledge via SKILL.md), plugins/connectors (MCP), and subagents (separating the maker from the checker) — and shows they now exist almost identically in both the Codex app and Claude Code (e.g. both ship a /goal "run-until-verifiable-condition" loop where a fresh model grades completion). His caution: verification, comprehension debt and "cognitive surrender" all get sharper, not easier — "Build the loop. Stay the engineer."

Notable Themes

Cross-cutting observations

  • The labs are converging on AI-for-cyber-defense — with governments in the loop. OpenAI's Daybreak is the third major lab move in a month after Anthropic's defending-code-reference-harness + MITRE ATT&CK mapping and AWS's GA frontier security agents. All three pair frontier capability with access controls and national-government partnerships (OpenAI alone named Australia, Canada, France, Germany, Japan, Korea, the EU/ENISA and the UK, plus CAISI testing). The competitive frontier in security is no longer "can the model find bugs" but "can we ship fixes responsibly at scale without arming attackers."
  • "Find" is solved; "fix" is the new bottleneck. OpenAI states it explicitly — defenders are now overwhelmed by the volume of AI-found vulnerabilities, so Codex Security and Patch the Planet target validation, patch generation and maintainer capacity. This reframes a year of "AI finds vulns faster" anxiety into a remediation-throughput problem, and explains why Trail of Bits/HackerOne human review sits at the center.
  • Prompt injection remains structurally unsolved. The "role confusion" research Simon flags shows models weight style over content, so a benign-looking restyle of an attack drops success from 61% to 10% — and injection stays whack-a-mole "until LLMs achieve genuine role perception." It's a sobering counterweight on the same day OpenAI ships GPT‑5.5‑Cyber: the offense side of LLM security is still wide open.
  • "Loop engineering" is crystallizing as the successor to prompt engineering. Addy Osmani's piece (quoting Anthropic's Boris Cherny), OpenAI's own "Codex-Maxxing for Long-Running Work," and Simon's parallel-agent Moebius port all point the same way: stop hand-prompting, design self-running loops with a maker/checker split. Tellingly, the Codex app and Claude Code now expose the same primitives (automations, worktrees, skills, MCP connectors, subagents, /goal) — the tool wars are commoditizing into a shared shape.
  • Small models keep beating their weight class. GLM‑5.2 running locally, a 3B VibeThinker claiming to top Opus 4.5 on reasoning, and a 0.2B Moebius model doing 10B-level inpainting in a browser all trended on the same day. The efficiency frontier (small/local/edge) is generating as much practitioner energy as the frontier-model releases.
